# Contributing

Thanks for your interest in improving `tailor_resume_for_role`!

## Ways to Contribute
- File issues: bugs, prompt edge cases, feature requests.
- Improve docs: clarify wording, add examples, fix typos.
- Add role examples: realistic JDs/resumes and expected outputs.
- Enhance templates: resume/rationale structures and style guidance.
- Testing/validation: checks for ATS headings, bullet quality, and no-fabrication rules.

## Pull Request Guidelines
- Fork the repo and create a feature branch (`feat/…`, `fix/…`, `docs/…`).
- Keep changes focused and minimal; one concern per PR.
- Update or add docs/examples/test_cases if behavior changes.
- Ensure Markdown renders cleanly (headings, lists, code fences).
- Write clear PR descriptions: problem, approach, trade-offs.

## Style & Principles
- Preserve truth. No fake jobs, tools, metrics, or inflated scope.
- Optimize for strategic tailoring over generic rewriting.
- Favor clarity, specificity, and scanability.
- Avoid keyword stuffing and buzzword chains.

## Development Setup (optional)
- If you add a CLI later, include setup steps in `scripts/` and update README.
- Prefer simple, dependency-light tooling.

## Code of Conduct
- Be respectful, constructive, and collaborative. Assume good intent.
